# Entity Migration Summary

## Overview
This document summarizes the migration from in-memory storage to JPA/MySQL persistence for the UXRay backend.

## What Was Changed

### 1. Created New Entities

#### User Entity (`/backend/src/main/java/com/dryno/backend/entity/User.java`)
- **Purpose**: Manage user authentication and Google OAuth tokens
- **Fields**:
  - `id` (UUID): Primary key
  - `email`: User's email address (unique)
  - `name`: User's display name
  - `googleId`: Google OAuth ID (unique)
  - `pictureUrl`: User's profile picture
  - `refreshToken`: OAuth refresh token
  - `accessToken`: OAuth access token
  - `tokenExpiry`: Token expiration timestamp
  - `createdAt`, `updatedAt`: Audit timestamps
- **Table**: `users`

### 2. Converted Domain Classes to JPA Entities

#### Job Entity (`/backend/src/main/java/com/dryno/backend/domain/Job.java`)
- **Changes**:
  - Added `@Entity` and `@Table(name = "jobs")` annotations
  - Added JPA field annotations (`@Id`, `@Column`, `@Enumerated`, etc.)
  - Changed `screenshots` from synchronized list to `@OneToMany` relationship
  - Embedded `DesignAnalysisResult` and `Ga4Results` using `@Embedded`
  - Stored `detectedAbPlatforms` as JSON using `@JdbcTypeCode(SqlTypes.JSON)`
  - Added `@PreUpdate` lifecycle callback for automatic timestamp updates
  - Removed `final` modifiers to support JPA requirements
  - Added no-arg constructor for JPA
  - Updated setter methods to manually update `updatedAt` timestamp

#### Screenshot Entity (`/backend/src/main/java/com/dryno/backend/domain/Screenshot.java`)
- **Changes**:
  - Added `@Entity` and `@Table(name = "screenshots")` annotations
  - Added `@ManyToOne` relationship with Job
  - Added JPA field annotations
  - Added no-arg constructor for JPA
  - Added setter methods for JPA field access
  - Removed `final` modifiers

#### DesignAnalysisResult (`/backend/src/main/java/com/dryno/backend/domain/DesignAnalysisResult.java`)
- **Changes**:
  - Added `@Embeddable` annotation (embedded in Job)
  - Stored `keyPoints` list as JSON using `@JdbcTypeCode(SqlTypes.JSON)`
  - Added null safety checks in getter methods
  - Changed field initialization to support JPA

#### Ga4Results (`/backend/src/main/java/com/dryno/backend/domain/Ga4Results.java`)
- **Changes**:
  - Added `@Embeddable` annotation (embedded in Job)
  - Stored all list fields as JSON using `@JdbcTypeCode(SqlTypes.JSON)`
  - Added JPA column annotations with descriptive names
  - Added null safety checks in all getter methods
  - Updated replace methods to handle null cases properly

### 3. Converted Repositories to JPA

#### JobRepository (`/backend/src/main/java/com/dryno/backend/repository/JobRepository.java`)
- **Before**: Custom class with ConcurrentHashMap for in-memory storage
- **After**: JPA interface extending `JpaRepository<Job, UUID>`
- **New Methods**:
  - `findByStatus(JobStatus status)`
  - `findByStatusOrderByCreatedAtDesc(JobStatus status)`
  - `findAllByOrderByCreatedAtDesc()`
- **Removed**: All custom storage logic (handled by Spring Data JPA)

#### UserRepository (`/backend/src/main/java/com/dryno/backend/repository/UserRepository.java`)
- **New JPA Repository**: Interface extending `JpaRepository<User, UUID>`
- **Methods**:
  - `findByEmail(String email)`
  - `findByGoogleId(String googleId)`
  - `existsByEmail(String email)`

### 4. Created New Services

#### UserService (`/backend/src/main/java/com/dryno/backend/service/UserService.java`)
- **Purpose**: Business logic for user management
- **Key Methods**:
  - `createUser(User user)`: Create a new user
  - `getUserById(UUID id)`: Get user by ID
  - `getUserByEmail(String email)`: Get user by email
  - `getUserByGoogleId(String googleId)`: Get user by Google ID
  - `getAllUsers()`: Get all users
  - `updateUser(UUID id, User updatedUser)`: Update user details
  - `deleteUser(UUID id)`: Delete user
  - `findOrCreateUser(...)`: Find existing or create new user (for OAuth)
  - `updateTokens(...)`: Update OAuth tokens
  - `existsByEmail(String email)`: Check if email exists

### 5. Created New Controllers

#### UserController (`/backend/src/main/java/com/dryno/backend/web/UserController.java`)
- **Purpose**: REST API endpoints for user CRUD operations
- **Endpoints**:
  - `POST /api/users`: Create user
  - `GET /api/users/{id}`: Get user by ID
  - `GET /api/users/email/{email}`: Get user by email
  - `GET /api/users`: Get all users
  - `PUT /api/users/{id}`: Update user
  - `DELETE /api/users/{id}`: Delete user
  - `GET /api/users/exists/{email}`: Check if email exists

## Database Schema

### Tables Created (Auto-generated by Hibernate)

#### `users` table
```sql
CREATE TABLE users (
    id BINARY(16) PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(255),
    google_id VARCHAR(255) UNIQUE,
    picture_url VARCHAR(500),
    refresh_token VARCHAR(1000),
    access_token VARCHAR(2000),
    token_expiry DATETIME,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL

```

#### `jobs` table

#### `screenshots` table
```sql
CREATE TABLE screenshots (
    id VARCHAR(255) PRIMARY KEY,
    page_url VARCHAR(2000),
    image_storage_path VARCHAR(500),
    created_at DATETIME NOT NULL,
    job_id BINARY(16),
    FOREIGN KEY (job_id) REFERENCES jobs(id)
);
```

## Key Technical Decisions

1. **JSON Storage**: Used `@JdbcTypeCode(SqlTypes.JSON)` from Hibernate Types to store complex lists/objects as JSON in MySQL
2. **Embedded Objects**: Used `@Embeddable` for DesignAnalysisResult and Ga4Results to keep them in the same table as Job
3. **UUID Keys**: Maintained UUID primary keys for Job and User entities
4. **Cascade Operations**: OneToMany relationship from Job to Screenshot uses `CASCADE.ALL` and `orphanRemoval=true`
5. **Lazy Loading**: Screenshots are loaded lazily to improve performance
6. **Automatic Timestamps**: Use `@PreUpdate` callback and manual updates to maintain `updatedAt` field

## Migration Path

### Before (In-Memory)
```java
@Repository
public class JobRepository {
    private final ConcurrentMap<UUID, Job> storage = new ConcurrentHashMap<>();
    // ... custom methods
}
```

### After (JPA/MySQL)
```java
@Repository
public interface JobRepository extends JpaRepository<Job, UUID> {
    // Spring Data JPA auto-implements CRUD
    // Custom query methods defined by naming convention
}
```

## Benefits

1. **Data Persistence**: Data survives application restarts
2. **Transaction Support**: ACID guarantees for data operations
3. **Query Optimization**: Database-level optimizations
4. **Scalability**: Can handle much larger datasets
5. **Relationships**: Proper foreign key relationships between entities
6. **Standard CRUD**: Spring Data JPA provides all standard operations out of the box
7. **Complex Queries**: Can leverage SQL for complex analytics
